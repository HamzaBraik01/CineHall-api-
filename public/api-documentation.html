CineHall API Documentation

Welcome to the CineHall API documentation. This guide will help you understand and integrate with our cinema management system.

Authentication

Login

Endpoint: POST /api/login

Request Body:

{
    "email": "user@example.com",
    "password": "password123"
}

Response:

{
    "token": "your_jwt_token",
    "user": {
        "id": 1,
        "name": "John Doe",
        "email": "user@example.com"
    }
}

Register

Endpoint: POST /api/register

Request Body:

{
    "name": "John Doe",
    "email": "user@example.com",
    "password": "password123",
    "phone": "1234567890"
}

Movies

Get All Movies

Endpoint: GET /api/movies

Response:

{
    "id": 1,
    "title": "Movie Title",
    "description": "Movie Description",
    "duration": 120,
    "genre": "Action",
    "release_date": "2024-03-20",
    "poster_url": "path/to/poster.jpg"
}

Get Movie Details

Endpoint: GET /api/movies/{id}

Sessions

Get All Sessions

Endpoint: GET /api/sessions

Response:

{
    "id": 1,
    "movie_name": "Movie Title",
    "hall_name": "Hall 1",
    "start_time": "2024-03-20 10:00:00",
    "language": "EN",
    "type": "normal"
}

Get Session Details

Endpoint: GET /api/sessions/{id}

Halls

Get All Halls

Endpoint: GET /api/halls

Response:

{
    "id": 1,
    "name": "Hall 1",
    "capacity": 100,
    "type": "VIP"
}

Seats

Create Seats

Endpoint: POST /api/seats

Request Body:

{
    "hall_id": 1,
    "start_number": 1,
    "end_number": 10,
    "status": "available"
}

Response:

{
    "message": "Seats created successfully",
    "data": [
        {
            "id": 1,
            "hall_id": 1,
            "seat_number": 1,
            "status": "available"
        }
        // ... more seats
    ]
}

Get Seats by Hall

Endpoint: GET /api/seats/hall/{hallId}

Reservations

Create Reservation

Endpoint: POST /api/reservations/add/{session_id}/{seat_ids}

Request Body:

{
    "status": "pending"
}

Response:

{
    "id": 1,
    "user_id": 1,
    "session_id": 1,
    "status": "pending",
    "seats": [...],
    "session": {...},
    "user": {...},
    "tickets": [...]
}

Get User Reservations

Endpoint: GET /api/reservations/user/{userId}

Error Handling

The API uses standard HTTP response codes:

  • 200 - Success
  • 201 - Created
  • 400 - Bad Request
  • 401 - Unauthorized
  • 404 - Not Found
  • 500 - Server Error
